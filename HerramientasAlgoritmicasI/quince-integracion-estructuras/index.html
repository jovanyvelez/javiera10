⚡ La magia de combinar

🔀

Una sola estructura

Solo if sin más: decides qué hacer, pero solo una vez. Solo arreglos sin if: tienes datos pero no los procesas. Solo ciclos: repites lo mismo sin pensar.

Las 3 juntas

Un arreglo con un ciclo que recorre y un if que decide: eso es el 90% de los programas reales. Hoy lo vas a dominar.

🧰 El Recap: 3 Patrones Universales

⏱️ 45 minutos · Las 3 combinaciones que vas a usar toda tu carrera

🔍 ¿Cómo se combinan las 3 estructuras?

Cuando combinas arreglo + ciclo + if, obtienes 3 patrones que aparecen en casi todo programa real. Si los dominas, puedes resolver el 90% de los problemas.

📐 Patrón 1: Buscar

🔎 Recorrer y encontrar

Recorres un arreglo con un ciclo y, dentro, usas un if para ver si encontraste lo que buscas. Si lo encuentras, lo guardas o activas una bandera.

// Buscar si existe un nombre en una lista bool Existe(string[] nombres, string buscado) { for (int i = 0; i < nombres.Length; i++) { if (nombres[i] == buscado) // ← LA COMBINACIÓN { return true; // ¡encontrado! } } return false; }

📐 Patrón 2: Contar / Acumular

📊 Recorrer y sumar

Igual que buscar, pero en vez de retornar al primer match, recorres TODO y acumulas (un contador, una suma, un promedio).

// Contar cuántos aprobados hay int ContarAprobados(double[] notas) { int contador = 0; for (int i = 0; i < notas.Length; i++) { if (notas[i] >= 60) // ← EL IF DENTRO DEL CICLO { contador++; } } return contador; }

📐 Patrón 3: Transformar

🔄 Recorrer y crear uno nuevo

Recorres un arreglo y construyes uno nuevo basado en una condición. Por ejemplo: de una lista de notas, sacar solo las aprobadas.

// Obtener solo las notas aprobadas double[] SoloAprobadas(double[] notas) { double[] resultado = new double[100]; int cant = 0; for (int i = 0; i < notas.Length; i++) { if (notas[i] >= 60) // ← MISMO IF, OTRO RESULTADO { resultado[cant] = notas[i]; cant++; } } return resultado; }

🛒 Mini-Proyecto: Tienda de Frutas

⏱️ 45 minutos · Tu primer sistema completo: catálogo + carrito + descuentos

🎯 El reto

Construir una tienda que: (1) muestre un catálogo de frutas (arreglo), (2) permita agregar productos al carrito (acumulador), y (3) aplique un descuento del 10% si el total supera los $50 (if). Todo en un solo programa.

🏗️ Arquitectura del programa

📦

Arreglo: catálogo

Las 8 frutas con sus nombres y precios. string[] frutas y double[] precios.

🔁

Ciclo: el menú

Un while que muestra el catálogo y permite agregar productos hasta que el usuario diga "salir".

🔀

If: el descuento

Si el total > 50, aplica 10% de descuento. Si no, precio normal.

🎮 Simulador: Tienda interactiva

Haz clic en cada fruta para agregarla al carrito. Verás el total actualizándose en tiempo real. Si superas los $50, el descuento se aplica solo.

💻 El código (simplificado)

double total = 0; int items = 0; double LIMITE = 50; double DESC = 0.10; // Por cada producto agregado: items++; total += precioElegido; if (total > LIMITE) { // aplicar 10% de descuento }

💻 El código completo

TiendaFrutas.cs
using System;

class TiendaFrutas
{
    static void Main()
    {
        // ══════════════════════════════════════
        // 1. CATÁLOGO (arreglos paralelos)
        // ══════════════════════════════════════
        string[] frutas = {
            "Manzana", "Banana", "Naranja", "Pera",
            "Uvas",    "Fresa",  "Kiwi",    "Sandía"
        };
        double[] precios = { 2.50, 1.80, 3.00, 2.80,
                             5.50, 4.20, 6.00, 8.00 };

        // ══════════════════════════════════════
        // 2. VARIABLES DE CONTROL
        // ══════════════════════════════════════
        double subtotal = 0;
        int    items    = 0;
        const double LIMITE_DESCUENTO = 50.0;
        const double PORCENTAJE       = 0.10;  // 10%

        Console.WriteLine("🛒 Bienvenido a la Frutería Don Código");
        Console.WriteLine("Escribe 'salir' cuando termines de comprar\n");

        // ══════════════════════════════════════
        // 3. BUCLE PRINCIPAL (ciclo)
        // ══════════════════════════════════════
        while (true)
        {
            Console.WriteLine("\n════ CATÁLOGO ════");
            for (int i = 0; i < frutas.Length; i++)
                Console.WriteLine($"{i + 1}. {frutas[i],-10} ${precios[i]:F2}");

            Console.Write("\nElige una fruta (1-8) o 'salir': ");
            string opcion = Console.ReadLine();

            if (opcion.ToLower() == "salir")
                break;

            if (!int.TryParse(opcion, out int num) || num < 1 || num > frutas.Length)
            {
                Console.WriteLine("⚠️ Opción inválida");
                continue;
            }

            // ══════════════════════════════════════
            // 4. ACUMULAR (acumulador)
            // ══════════════════════════════════════
            int idx = num - 1;
            subtotal += precios[idx];
            items++;
            Console.WriteLine($"✓ {frutas[idx]} añadido. Subtotal: ${subtotal:F2}");
        }

        // ══════════════════════════════════════
        // 5. CALCULAR DESCUENTO (if)
        // ══════════════════════════════════════
        double descuento = 0;
        if (subtotal > LIMITE_DESCUENTO)
        {
            descuento = subtotal * PORCENTAJE;
            Console.WriteLine($"\n🎉 ¡DESCUENTO del 10% aplicado! -${descuento:F2}");
        }

        double total = subtotal - descuento;

        // ══════════════════════════════════════
        // 6. MOSTRAR RESUMEN
        // ══════════════════════════════════════
        Console.WriteLine("\n════ TICKET ════");
        Console.WriteLine($"Items:    {items}");
        Console.WriteLine($"Subtotal: ${subtotal:F2}");
        if (descuento > 0)
            Console.WriteLine($"Descuento: -${descuento:F2}");
        Console.WriteLine($"TOTAL:    ${total:F2}");
    }
}

🎓 Las 3 estructuras trabajando juntas

En este programa ves: arreglos para el catálogo, while + for para mostrar y repetir, y if para el descuento. Esas 20 líneas de lógica son el esqueleto del 90% de los sistemas de venta del mundo real.

💡 Truco del profesor

Cuando veas un problema, pregúntate: "¿qué arreglo uso, qué ciclo lo recorre, y qué if decide?" Si puedes responder esas 3 preguntas, tienes el 80% del problema resuelto.

🎲 Mini-Proyecto: Adivina el Número

⏱️ 45 minutos · El clásico juego con ranking persistente

🎯 El reto

La computadora piensa un número del 1 al 100. El usuario tiene que adivinarlo. Por cada intento, la app le dice "más alto" o "más bajo". Al final, guarda cuántos intentos necesitó en un ranking (arreglo).

🧠 Las 3 estructuras en acción

📦

Arreglo: ranking

int[] mejoresIntentos = new int[5] guarda los 5 mejores puntajes en orden ascendente.

🔁

Ciclo: reintentos

Un while (intento != secreto) que sigue pidiendo hasta que el usuario acierte.

🔀

If: pistas

if (intento > secreto) → "Más bajo". if (intento < secreto) → "Más alto".

🎮 Simulador: ¡Adivina!

La app pensó un número entre 1 y 100. Escribe tu intento y presiona "Probar". Te dirá si es más alto o más bajo. Intenta hacerlo en la menor cantidad de intentos para entrar al ranking.

💻 El código completo

AdivinaNumero.cs
using System;

class AdivinaNumero
{
    static void Main()
    {
        Random rnd = new Random();
        int secreto = rnd.Next(1, 101);  // 1 a 100

        // ══════════════════════════════════════
        // RANKING: los 5 mejores puntajes
        // ══════════════════════════════════════
        int[] ranking = new int[5] { 999, 999, 999, 999, 999 };
        int intentos = 0;
        int intento;

        Console.WriteLine("🎲 Pensé un número del 1 al 100. ¡Adivínalo!\n");

        // ══════════════════════════════════════
        // CICLO: reintentar hasta acertar
        // ══════════════════════════════════════
        do
        {
            Console.Write("Tu intento: ");
            while (!int.TryParse(Console.ReadLine(), out intento)
                   || intento < 1 || intento > 100)
            {
                Console.Write("⚠️ Ingresa un número entre 1 y 100: ");
            }

            intentos++;

            // ══════════════════════════════════════
            // IF: pista según comparación
            // ══════════════════════════════════════
            if (intento > secreto)
                Console.WriteLine("📉 Más BAJO...");
            else if (intento < secreto)
                Console.WriteLine("📈 Más ALTO...");
            else
                Console.WriteLine($"\n🎉 ¡ACERTASTE en {intentos} intentos!");

        } while (intento != secreto);

        // ══════════════════════════════════════
        // GUARDAR EN RANKING
        // ══════════════════════════════════════
        for (int i = 0; i < ranking.Length; i++)
        {
            if (intentos < ranking[i])
            {
                // Mover los demás hacia abajo
                for (int j = ranking.Length - 1; j > i; j--)
                    ranking[j] = ranking[j - 1];
                ranking[i] = intentos;
                break;
            }
        }

        // Mostrar ranking
        Console.WriteLine("\n════ RANKING TOP 5 ════");
        for (int i = 0; i < ranking.Length; i++)
            Console.WriteLine($"{i + 1}. {ranking[i]} intentos");
    }
}

⚠️ El truco del ranking

Para insertar un nuevo puntaje en el arreglo, mueves los peores hacia abajo con un for que va al revés. Es un patrón clásico de "insertar en orden" — lo vas a ver en bases de datos, listas de espera, y más.

💡 ¿Por qué <= y >= en lugar de < y >?

Porque el 1 y el 100 son válidos. Si usas <, rechazas el 1. La condición intento >= 1 && intento <= 100 incluye los extremos. Detalles como este separan un programa junior de uno pro.

🗳️ Mini-Proyecto: Sistema de Votación

⏱️ 45 minutos · 4 candidatos, conteo en vivo, encuentra al ganador

🎯 El reto

4 candidatos compiten. Los usuarios emiten votos (uno a la vez). El sistema debe: (1) contar votos por candidato (arreglo + acumulador), (2) mostrar estadísticas en vivo (porcentajes), y (3) declarar un ganador al final (if con la mayor cantidad de votos).

🏛️ Las 3 estructuras en acción

📦

Arreglo: candidatos y votos

Dos arreglos paralelos: string[] candidatos = {...} y int[] votos = new int[4] (inicia en 0 cada uno).

🔁

Ciclo: votar varias veces

Un while que pide votos hasta que el usuario escriba "fin". Cada voto se procesa y se acumula.

🔀

If: encontrar al ganador

Recorrer el arreglo votos y comparar para encontrar el máximo. Quien tenga más, gana.

🎮 Simulador: Votación en vivo

Haz clic en cada candidato para emitir un voto. Verás la barra y los contadores actualizándose en tiempo real. Cuando termines, presiona "Cerrar votación" para ver al ganador.

🏆 ¡GANADOR!

💻 El código completo

SistemaVotacion.cs
using System;

class SistemaVotacion
{
    static void Main()
    {
        // ══════════════════════════════════════
        // 1. CANDIDATOS (arreglos paralelos)
        // ══════════════════════════════════════
        string[] candidatos = {
            "🤖 Atlas",
            "🎨 Pixel",
            "🔬 Nova",
            "🌟 Cosmos"
        };
        int[] votos = new int[4];  // todos inician en 0

        Console.WriteLine("🗳️ Elecciones del Colegio 2026");
        Console.WriteLine("Vota por tu candidato favorito. Escribe 'fin' para terminar.\n");

        // ══════════════════════════════════════
        // 2. BUCLE DE VOTACIÓN
        // ══════════════════════════════════════
        while (true)
        {
            // Mostrar candidatos
            for (int i = 0; i < candidatos.Length; i++)
                Console.WriteLine($"{i + 1}. {candidatos[i]} ({votos[i]} votos)");

            Console.Write("\nTu voto (1-4) o 'fin': ");
            string op = Console.ReadLine();

            if (op.ToLower() == "fin") break;

            if (!int.TryParse(op, out int n) || n < 1 || n > candidatos.Length)
            {
                Console.WriteLine("⚠️ Voto inválido");
                continue;
            }

            // ══════════════════════════════════════
            // 3. ACUMULAR EL VOTO
            // ══════════════════════════════════════
            votos[n - 1]++;
            Console.WriteLine($"✓ Voto registrado para {candidatos[n - 1]}");
        }

        // ══════════════════════════════════════
        // 4. ENCONTRAR AL GANADOR (buscar el máximo)
        // ══════════════════════════════════════
        int idxGanador = 0;
        for (int i = 1; i < votos.Length; i++)
        {
            if (votos[i] > votos[idxGanador])
                idxGanador = i;
        }

        // ══════════════════════════════════════
        // 5. MOSTRAR RESULTADOS
        // ══════════════════════════════════════
        int totalVotos = 0;
        foreach (int v in votos) totalVotos += v;

        Console.WriteLine("\n════ RESULTADOS ════");
        for (int i = 0; i < candidatos.Length; i++)
        {
            double porcentaje = totalVotos > 0 ? (votos[i] * 100.0 / totalVotos) : 0;
            Console.WriteLine($"{candidatos[i],-15} {votos[i],3} votos  ({porcentaje,5:F1}%)");
        }

        Console.WriteLine($"\n🏆 GANADOR: {candidatos[idxGanador]} con {votos[idxGanador]} votos");
    }
}

🎓 El patrón "buscar el máximo"

Para encontrar al ganador: empiezas suponiendo que es el primero (idxGanador = 0), luego recorres todos los demás y, si encuentras uno con más votos, lo reemplazas. Al final tienes el índice del mayor. Es el mismo patrón para encontrar el más alto, el más bajo, o el más caro.

📝 Evaluación Formativa + Taller

⏱️ 45 minutos · Una evaluación sin nota: para que sepas qué dominas y qué reforzar

🎯 ¿Qué es una evaluación formativa?

Es una evaluación sin nota numérica. Su objetivo no es ponerte un 5 o un 3: es que tú mismo sepas qué conceptos dominas y cuáles necesitas repasar. Cada respuesta tiene feedback inmediato que te explica por qué. Si te equivocas, no pasa nada: vuelves, lees, y mejoras.

📋 La evaluación: 4 preguntas de integración

Estas preguntas mezclan todo lo aprendido en la forja. Lee con calma, piensa, y elige la mejor respuesta.

📝 Evaluación Formativa — Integración Total

1. Tienes un programa que recorre un arreglo de notas y cuenta cuántas son mayores a 60. ¿Qué patrón estás usando?

Un ciclo for recorre el arreglo, y dentro un if (nota >= 60) incrementa el contador. Esa es la combinación clásica "arreglo + ciclo + if".

2. En el código foreach (var item in productos) { if (item.Precio > 100) Console.WriteLine(item.Nombre); }, ¿qué hace este programa?

Console.WriteLine solo IMPRIME el nombre, no acumula ni cuenta. Es un patrón "mostrar/filtrar": recorre con foreach y filtra con if.

3. Para encontrar al CANDIDATO GANADOR de una votación con 4 candidatos, ¿cuál es la mejor estrategia?

El patrón "buscar el máximo": recorres con for y comparas cada valor contra el que llevas como "ganador actual". Al final, ese índice es el ganador.

4. ¿Por qué un programa con solo if y arreglos, pero SIN ciclos, es muy limitado?

Sin ciclos, solo podrías aplicar el if a UN elemento del arreglo. Los ciclos son los que permiten revisar todos los elementos automáticamente.
