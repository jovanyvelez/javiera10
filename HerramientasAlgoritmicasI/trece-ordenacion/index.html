💧 Método Burbuja

⏱️ 45 minutos · El más intuitivo: como burbujas subiendo en un vaso de refresco

🧠 ¿Cómo funciona?

Imagina un vaso de refresco. Las burbujas más grandes suben más rápido que las pequeñas. En Bubble Sort, los números grandes "burbujean" hacia el final del arreglo a través de intercambios sucesivos entre vecinos.

  1. Compara dos elementos vecinos (el de la izquierda con el de la derecha).
  2. Si el izquierdo es mayor que el derecho → intercámbialos (swap).
  3. Avanza al siguiente par y repite hasta el final del arreglo.
  4. Al terminar una pasada, el número más grande ya está en su posición final. Una burbuja llegó a la cima 🫧
  5. Repite con el resto del arreglo, ignorando los ya ordenados.

💻 Código en C#

Aquí está la implementación completa. Fíjate en los dos bucles anidados: el externo controla las pasadas, el interno compara vecinos.

BubbleSort.cs
using System;

class BubbleSort
{
    static void Main()
    {
        int[] arr = { 64, 34, 25, 12, 22, 11, 90 };

        Console.WriteLine("Arreglo original:");
        Imprimir(arr);

        OrdenarBurbuja(arr);

        Console.WriteLine("\nArreglo ordenado:");
        Imprimir(arr);
    }

    static void OrdenarBurbuja(int[] arr)
    {
        int n = arr.Length;

        // Bucle externo: una pasada por cada elemento
        for (int i = 0; i < n - 1; i++)
        {
            // Bucle interno: compara vecinos
            // Restamos 'i' porque los últimos 'i' ya están ordenados
            for (int j = 0; j < n - i - 1; j++)
            {
                // Si el actual es mayor que el siguiente → swap
                if (arr[j] > arr[j + 1])
                {
                    int temp   = arr[j];
                    arr[j]     = arr[j + 1];
                    arr[j + 1] = temp;
                }
            }
        }
    }

    static void Imprimir(int[] arr)
    {
        foreach (int x in arr)
            Console.Write(x + " ");
        Console.WriteLine();
    }
}

⚡ Versión optimizada (con bandera)

Si en una pasada no hubo ningún intercambio, el arreglo ya está ordenado y podemos detenernos antes. Esto se llama early exit y mejora el mejor caso a O(n).

BubbleSort_Optimizado.cs
static void OrdenarBurbujaOpt(int[] arr)
{
    int n = arr.Length;
    bool huboSwap;

    for (int i = 0; i < n - 1; i++)
    {
        huboSwap = false;

        for (int j = 0; j < n - i - 1; j++)
        {
            if (arr[j] > arr[j + 1])
            {
                int temp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = temp;
                huboSwap = true; // ¡Marcamos que sí hubo intercambio!
            }
        }

        // Si en esta pasada NO hubo swap → ya está ordenado
        if (!huboSwap)
        {
            Console.WriteLine($"¡Optimización! Terminó en {i + 1} pasadas.");
            break;
        }
    }
}

💡 Complejidad de Burbuja

Peor caso: O(n²) — cuando el arreglo está invertido.
Mejor caso: O(n) — con la optimización, si ya está ordenado.
Promedio: O(n²).
Espacio: O(1) — solo usa una variable temporal.

🎯 Método de Selección

⏱️ 45 minutos · El cazador del mínimo: siempre encuentra al más pequeño

🧠 La idea

Imagina que tienes que ordenar a tus amigos por estatura. ¿Qué harías? Miras a todos, encuentras al más bajito, y lo colocas al inicio. Luego buscas al segundo más bajito entre los restantes y lo pones después. Y así sucesivamente. Eso es Selection Sort.

  1. Divide el arreglo en dos partes: ordenada (al inicio, vacía) y desordenada (todo el resto).
  2. Busca el mínimo en la parte desordenada.
  3. Intercámbialo con el primer elemento de la parte desordenada.
  4. Mueve la frontera: ahora ese elemento forma parte de la zona ordenada. Repite.

💻 Código en C#

Nota el detalle clave: primero buscamos el índice del mínimo, y solo al final hacemos UN swap por pasada.

SelectionSort.cs
using System;

class SelectionSort
{
    static void Main()
    {
        int[] arr = { 64, 25, 12, 22, 11 };

        Console.WriteLine("Antes: " + string.Join(", ", arr));
        OrdenarSeleccion(arr);
        Console.WriteLine("Después: " + string.Join(", ", arr));
    }

    static void OrdenarSeleccion(int[] arr)
    {
        int n = arr.Length;

        for (int i = 0; i < n - 1; i++)
        {
            // Suponemos que el mínimo es el primero de la zona desordenada
            int indiceMin = i;

            // Buscamos el verdadero mínimo en la zona desordenada
            for (int j = i + 1; j < n; j++)
            {
                if (arr[j] < arr[indiceMin])
                {
                    indiceMin = j; // ¡Encontramos uno más pequeño!
                }
            }

            // Si el mínimo NO es el actual, los intercambiamos
            if (indiceMin != i)
            {
                int temp    = arr[i];
                arr[i]      = arr[indiceMin];
                arr[indiceMin] = temp;
            }
        }
    }
}

⚠️ Diferencia clave con Burbuja

Burbuja puede hacer muchos swaps por pasada. Selección hace máximo UN swap por pasada. Por eso Selección suele ser más eficiente cuando los intercambios son costosos (por ejemplo, con estructuras grandes en memoria).

🃏 Método de Inserción

⏱️ 45 minutos · Como ordenar cartas en tu mano

🧠 La mecánica

¿Has jugado cartas? Cuando te reparten una carta nueva, la miras y la insertas en el lugar correcto de tu mano (que ya está ordenada). Desplazas las mayores hacia la derecha y metes la nueva donde va. ¡Eso es Insertion Sort en la vida real!

  1. El primer elemento ya se considera "ordenado".
  2. Toma el siguiente elemento (la llave o key).
  3. Compara la llave con los elementos ya ordenados, de derecha a izquierda.
  4. Cada elemento mayor que la llave se desplaza una posición a la derecha.
  5. Inserta la llave en el hueco que queda.

💻 Código en C#

InsertionSort.cs
using System;

class InsertionSort
{
    static void Main()
    {
        int[] arr = { 12, 11, 13, 5, 6 };

        Console.WriteLine("Antes: " + string.Join(", ", arr));
        OrdenarInsercion(arr);
        Console.WriteLine("Después: " + string.Join(", ", arr));
    }

    static void OrdenarInsercion(int[] arr)
    {
        int n = arr.Length;

        // Empezamos desde el segundo elemento (índice 1)
        for (int i = 1; i < n; i++)
        {
            int llave = arr[i];  // 🔑 La carta que vamos a insertar
            int j = i - 1;       // Índice del último ya ordenado

            // Desplazamos a la derecha los elementos mayores que la llave
            while (j >= 0 && arr[j] > llave)
            {
                arr[j + 1] = arr[j];
                j--;
            }

            // Insertamos la llave en su posición correcta
            arr[j + 1] = llave;
        }
    }
}

🏆 ¿Cuándo Insertion Sort es el REY?

Cuando el arreglo está casi ordenado (pocos elementos fuera de lugar). En ese caso, su rendimiento se acerca a O(n). Por eso muchos algoritmos avanzados (como Timsort, usado en Python y Java) usan Inserción como paso final.

🎮 ¿Qué es Big O?

Cuando hablamos de qué tan rápido es un algoritmo, usamos algo llamado Big O. Es una forma de medir cómo crece el trabajo a medida que crece el número de datos (n). No mide segundos, mide tendencia.

📊 Big O como niveles de dificultad

🟢 O(n) — Lineal

Si duplicas los datos, duplicas el trabajo. Como leer una lista completa una vez.

🔵 O(n log n) — Rápido

Los algoritmos "inteligentes" como Merge Sort o Quick Sort viven aquí.

🔴 O(n²) — Lento

Si duplicas los datos, cuadruplicas el trabajo. Aquí viven Burbuja, Selección e Inserción.

Algoritmo Mejor caso Promedio Peor caso Espacio Estable*
💧 Burbuja O(n) O(n²) O(n²) O(1) ✅ Sí
🎯 Selección O(n²) O(n²) O(n²) O(1) ❌ No
🃏 Inserción O(n) O(n²) O(n²) O(1) ✅ Sí

*Estable = si hay dos elementos iguales, mantienen su orden original.
