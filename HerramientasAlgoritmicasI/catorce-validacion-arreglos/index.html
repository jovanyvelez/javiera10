🚨 Los 5 Errores Clásicos

⏱️ 45 minutos · Los bugs que más cuesta encontrar (y más fácil es prevenir)

🎭 ¿Por qué tantos programas fallan?

El 90% de los errores con arreglos vienen de 5 patrones repetidos. Si los conoces, los puedes evitar siempre. Son como los 5 villanos de una película: ya sabes quiénes son, ya sabes cómo derrotarlos.

👾 Villano #1: Índice fuera de rango

💥 IndexOutOfRangeException

Tratas de acceder a una posición que no existe en el arreglo. El arreglo tiene 5 elementos (índices 0-4) y tú pides el índice 7.

int[] notas = { 85, 90, 78 }; // 3 elementos (0,1,2) Console.WriteLine(notas[7]); // 💥 ¡BOOM! No existe la posición 7

Solución: siempre valida que el índice esté entre 0 y arreglo.Length - 1.

👾 Villano #2: Arreglo vacío

💥 Acceder a un arreglo que no tiene elementos

Tu programa crea un arreglo pero nadie le metió datos. El arreglo "existe" pero su Length es 0.

int[] notas = new int[0]; // arreglo vacío Console.WriteLine(notas[0]); // 💥 No hay posición 0, no hay NADA

Solución: antes de acceder, verifica que arreglo.Length > 0.

👾 Villano #3: Tipo de dato incorrecto

💥 El usuario escribió "perro" en vez de 25

Tu variable espera un número, pero el usuario escribió texto. int.Parse("perro") explota.

Console.Write("Edad: "); int edad = int.Parse(Console.ReadLine()); // si escribe "perro" → 💥

Solución: usa int.TryParse() en vez de int.Parse(). Devuelve true/false en vez de explotar.

👾 Villano #4: Valor fuera de rango válido

💥 El usuario metió una edad negativa o un porcentaje de 200%

El dato técnicamente es válido (es un número), pero no tiene sentido en tu contexto.

edad = -50; // 💥 nadie tiene -50 años nota = 200; // 💥 una nota no puede ser 200% porcentaje = 1.5; // 💥 los % no superan 1.0

Solución: valida rangos lógicos. if (edad < 0 || edad > 120) rechazar().

👾 Villano #5: Modificar mientras recorres

💥 Borrar o agregar elementos durante un bucle

Tu for asume que el arreglo tiene 10 elementos, pero tú borraste uno a la mitad del recorrido. El índice se descontrola.

for (int i = 0; i < items.Length; i++) { if (items[i].EsInvalido) items.RemoveAt(i); // 💥 ¡acabas de cambiar items.Length! }

Solución: recorre hacia atrás, o usa una lista temporal, o usa Where() con LINQ.

🚧 Validador de Índices

⏱️ 45 minutos · El portero del arreglo: nada entra sin estar en la lista

🚪 Analogía: El portero del club

Imagina un club con 5 personas adentro (índices 0, 1, 2, 3, 4). El portero tiene una lista con esos 5 números. Si alguien llega con el número "7", el portero dice: "7 no está en la lista, no puedes entrar". Eso es validar índices.

📐 La regla de oro

Para acceder a un índice i de un arreglo arr:

0 <= i < arr.Length

El índice nunca puede ser negativo, y nunca puede ser igual o mayor que Length.

🛡️ El patrón de validación de 3 pasos

1️⃣

Paso 1: ¿El índice es válido?

Verifica que i >= 0 y que i < arr.Length. Si no, rechazas inmediatamente.

2️⃣

Paso 2: ¿El arreglo tiene datos?

Verifica que arr.Length > 0. Si está vacío, no hay nada que acceder.

3️⃣

Paso 3: Usa try-catch de red de seguridad

Aunque valides, usa try/catch como airbag por si acaso.

📝 Las dos formas: sin escudo y con escudo

❌ Sin validación (peligroso)

static int ObtenerNota(int[] notas, int i) { return notas[i]; // 💥 boom si i está fuera }

✅ Con validación (seguro)

static int ObtenerNota(int[] notas, int i) { if (i < 0 || i >= notas.Length) { Console.WriteLine("Índice fuera de rango"); return -1; // valor centinela } return notas[i]; // ahora sí es seguro }

💻 Función segura para obtener un elemento

ValidadorIndices.cs
using System;

class ValidadorIndices
{
    static void Main()
    {
        int[] notas = { 85, 90, 78, 92, 88 };

        // Probemos varios índices
        int[] pruebas = { -1, 0, 2, 5, 100 };

        foreach (int i in pruebas)
        {
            int resultado = ObtenerSeguro(notas, i);
            Console.WriteLine($"notas[{i}] = {resultado}");
        }
    }

    /// <summary>
    /// Obtiene un elemento del arreglo de forma SEGURA.
    /// Devuelve -1 si el índice es inválido.
    /// </summary>
    static int ObtenerSeguro(int[] arr, int i)
    {
        // ══════════════════════════════════════
        // VALIDACIÓN DE 3 PASOS
        // ══════════════════════════════════════
        if (arr == null)
        {
            Console.WriteLine("❌ El arreglo es null");
            return -1;
        }

        if (arr.Length == 0)
        {
            Console.WriteLine("❌ El arreglo está vacío");
            return -1;
        }

        if (i < 0 || i >= arr.Length)
        {
            Console.WriteLine($"❌ Índice {i} fuera de rango (0 a {arr.Length - 1})");
            return -1;
        }

        // Si llegamos aquí, es seguro acceder
        return arr[i];
    }
}

💡 El truco del "valor centinela"

Cuando la validación falla, devolvemos un valor imposible (como -1 para notas que van de 0-100). El que llama a la función sabe que -1 significa "no se pudo". Es mejor que lanzar una excepción porque te deja manejar el error con un if.

🔍 Validador de Valores

⏱️ 45 minutos · El inspector de calidad: nada se guarda sin estar en rango

🏭 Analogía: La fábrica de galletas

Imagina una fábrica de galletas. Antes de meter una galleta a la caja, un inspector verifica: ¿está bien horneada? ¿tiene el peso correcto? ¿no está rota?. Si falla cualquier revisión, no entra. Lo mismo hacemos con los datos antes de meterlos a un arreglo.

📋 Las 4 validaciones típicas

📏

1. Rango numérico

El número está entre el mínimo y máximo esperado.
if (edad >= 0 && edad <= 120)

📝

2. No vacío

El texto no es vacío ni solo espacios.
if (!string.IsNullOrWhiteSpace(nombre))

📏

3. Longitud del texto

El texto tiene el largo esperado (ni muy corto, ni muy largo).
if (nombre.Length >= 3 && nombre.Length <= 50)

🎯

4. Formato

El texto cumple un patrón (email, teléfono, código postal).
if (email.Contains("@"))

📝 Ejemplo: validar 4 campos de un estudiante

❌ Sin validación (acepta cualquier cosa)

// Esto acepta: nombre="" edad=-50 nota=200 estudiantes.Add(new Estudiante { Nombre = nombre, Edad = edad, // puede ser -50 💥 Nota = nota // puede ser 999 💥 });

✅ Con validación (rechaza datos malos)

if (EsValido(nombre, edad, nota, out string error)) { estudiantes.Add(new Estudiante { ... }); } else { Console.WriteLine("❌ " + error); } static bool EsValido(...) { if (string.IsNullOrWhiteSpace(nombre)) { error = "nombre vacío"; return false; } if (edad < 0 || edad > 120) { error = "edad fuera de rango"; return false; } if (nota < 0 || nota > 100) { error = "nota fuera de rango"; return false; } return true; }

⚠️ No valides "después"

El error clásico es: guardar el dato, hacer cálculos, y después validar. Si la validación falla, ya tienes basura en tu sistema. La validación debe ser al momento de la entrada — antes de tocar el arreglo.

🏰 El Validador Universal

⏱️ 45 minutos · Las 3 capas de defensa trabajando juntas

🛡️ Las 3 capas de un sistema a prueba de balas

Un programa profesional no se protege con una sola validación. Tiene 3 capas de defensa, como un castillo medieval: foso, muralla, y arqueros. Si una falla, la otra contiene el ataque.

🛡️

Capa 1: Validar entrada

Antes de tocar el arreglo: ¿el índice es válido? ¿el valor está en rango? ¿el texto no está vacío?

🔒

Capa 2: Validar estado

Antes de operar: ¿el arreglo tiene datos? ¿hay espacio? ¿el estado es coherente?

🪖

Capa 3: try-catch

Red de seguridad final. Si algo se escapa, lo capturamos y mostramos un mensaje amable.

🏗️ Sistema de registro de estudiantes completo

Aquí está el código profesional que combina las 3 capas. Léelo con calma — es un patrón que vas a usar toda tu carrera.

SistemaEstudiantes.cs
using System;
using System.Collections.Generic;

class SistemaEstudiantes
{
    // Arreglo con capacidad máxima de 100 estudiantes
    static string[] nombres = new string[100];
    static int[]    edades  = new int[100];
    static double[] notas   = new double[100];
    static int contador = 0;  // cuántos estudiantes hay realmente

    static void Main()
    {
        // Registramos varios estudiantes (algunos con datos malos a propósito)
        string[][] pruebas = {
            new[] {"Ana",      "16", "85"},   // ✓ válido
            new[] {"",         "20", "90"},   // ✗ nombre vacío
            new[] {"Pedro",    "-5", "75"},   // ✗ edad negativa
            new[] {"Lucía",    "17", "200"},  // ✗ nota fuera de rango
            new[] {"Carlos",   "150", "70"},  // ✗ edad > 120
            new[] {"Sofía",    "15", "92.5"}  // ✓ válido
        };

        foreach (var datos in pruebas)
        {
            if (RegistrarEstudiante(datos[0], int.Parse(datos[1]), double.Parse(datos[2])))
                Console.WriteLine($"✅ {datos[0]} registrado");
            else
                Console.WriteLine($"❌ {datos[0]} rechazado");
        }

        // Mostrar los que sí quedaron
        Console.WriteLine($"\n════ Total registrados: {contador} ════");
        for (int i = 0; i < contador; i++)
            Console.WriteLine($"{i + 1}. {nombres[i]} ({edades[i]} años) — Nota: {notas[i]}");
    }

    static bool RegistrarEstudiante(string nombre, int edad, double nota)
    {
        // ══════════════════════════════════════
        // CAPA 1: VALIDAR ENTRADA
        // ══════════════════════════════════════
        if (string.IsNullOrWhiteSpace(nombre))
        {
            Console.WriteLine($"   ⚠️ Nombre vacío");
            return false;
        }
        if (nombre.Length < 3 || nombre.Length > 50)
        {
            Console.WriteLine($"   ⚠️ Nombre debe tener entre 3 y 50 caracteres");
            return false;
        }
        if (edad < 0 || edad > 120)
        {
            Console.WriteLine($"   ⚠️ Edad {edad} fuera de rango (0-120)");
            return false;
        }
        if (nota < 0 || nota > 100)
        {
            Console.WriteLine($"   ⚠️ Nota {nota} fuera de rango (0-100)");
            return false;
        }

        // ══════════════════════════════════════
        // CAPA 2: VALIDAR ESTADO (arreglo)
        // ══════════════════════════════════════
        if (contador >= nombres.Length)
        {
            Console.WriteLine("   ⚠️ Arreglo lleno, no se puede registrar más");
            return false;
        }

        // ══════════════════════════════════════
        // CAPA 3: TRY-CATCH (red de seguridad)
        // ══════════════════════════════════════
        try
        {
            nombres[contador] = nombre;
            edades[contador]  = edad;
            notas[contador]   = nota;
            contador++;
            return true;
        }
        catch (Exception ex)
        {
            Console.WriteLine($"   ⚠️ Error inesperado: {ex.Message}");
            return false;
        }
    }
}

Este patrón se usa en toda la industria. Cada vez que recibas datos externos (de un usuario, de un archivo, de internet), aplícalo. Con el tiempo lo harás sin pensar.

💡 ¿Por qué 3 capas?

Si solo validas la entrada pero no usas try-catch, un bug interno puede tumbar tu app. Si solo usas try-catch, el usuario ve mensajes feos. Las 3 capas juntas: previenen, contienen, y comunican. Es la diferencia entre un junior y un senior.
